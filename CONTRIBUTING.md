## Кодекс поведения

Участвуя в проекте, ты соглашаешься соблюдать [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## С чего начать

1. Сделай форк репозитория
2. Создай ветку для своих изменений:
   ```bash
   git checkout -b feat/название-функции
   # или
   git checkout -b fix/название-бага
   ```
3. Внеси изменения
4. Убедись, что код соответствует [стилю](#стиль-кода)
5. Создай Pull Request

## Сообщить о баге

Перед тем как создавать Issue:
- Убедись, что баг ещё не описан в [открытых Issues](../../issues)
- Проверь, что используешь актуальную версию

Создай Issue с шаблоном **«Баг»** и заполни все поля.

## Предложить функцию

Создай Issue с шаблоном **«Запрос функции»** и опиши:
- Какую проблему решает функция
- Как ты видишь её реализацию

## Процесс Pull Request

1. Убедись, что к PR привязан соответствующий Issue
2. Заполни шаблон PR полностью
3. PR проходит ревью мейнтейнера перед мержем
4. Не пуш напрямую в `main` / `master`

### Соглашение об именах веток

| Тип | Формат |
|-----|--------|
| Новая функция | `feat/короткое-описание` |
| Исправление бага | `fix/короткое-описание` |
| Рефакторинг | `refactor/короткое-описание` |
| Документация | `docs/короткое-описание` |

## Стиль кода

- Длина строки — не более 88 символов
- Комментарии и docstring — на русском или английском (на твоё усмотрение)
- Все публичные функции должны иметь docstring
